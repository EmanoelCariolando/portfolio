## 📖 O que é este projeto?

Este repositório contém o código-fonte do **site de portfólio profissional** de Emanoel Cariolando.

**Atenção:** Este não é "mais um portfólio bonito".  
Ele foi construído com uma intenção muito clara e deliberada.

## 🎯 A Real Intenção do Projeto

O objetivo principal deste portfólio **não é apenas mostrar projetos**. É **demonstrar na prática** os valores que o desenvolvedor defende em toda a sua trajetória profissional:

> "Especialista em criar soluções de alta qualidade que melhoram processos e geram impacto real para empresas."

Isso fica evidente em vários aspectos:

- **Qualidade > quantidade**: O site preza por código limpo, consistente, acessível e fácil de manter, em vez de usar o máximo de tecnologias possível.
- **O portfólio é um dos projetos**: A própria landing page está listada na seção "Projetos", servindo como prova de conceito das habilidades em front-end.
- **Progressão de carreira intencional**: A seção de Experiência mostra uma jornada que vai de desenvolvedor/consultor → gerente de projetos (foco em processos e qualidade) → Full Stack Sênior (liderança técnica com foco em refatoração, testes e qualidade de código).
- **Sem over-engineering**: Mesmo dominando React, TypeScript, Spring Boot etc., o autor escolheu **HTML + CSS + JavaScript puro** para este projeto. Isso demonstra maturidade: usar a ferramenta certa para o problema (um site estático de apresentação não precisa de um framework pesado).

Em resumo: este portfólio é uma **declaração de princípios** — ele prova que o autor não só fala de qualidade, clean code, UX e acessibilidade, como aplica isso até no seu próprio cartão de visitas digital.

## 🛠️ Tecnologias e Abordagem

- **HTML5 semântico** + **CSS3 com Design System rigoroso**
- **JavaScript vanilla** (ES6+, sem frameworks)
- Design system aplicado de forma consistente (variáveis CSS, espaçamento em múltiplos de 8px, radius 4px, tipografia Inter, etc.)
- Foco forte em **acessibilidade**, **responsividade** e **performance**

O CSS possui comentários detalhados explicando a aplicação do Design System (inclusive referenciando um `design-system.md` que serviu de base).
